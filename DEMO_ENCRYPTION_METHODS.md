### Как показывать RSA

```
Карточка:
┌─────────────────────────────────┐
│ RSA-4096 ✓ Рекомендуется      │
│ (только для ключей)            │
├─────────────────────────────────┤
│ Описание: Асимметричный,       │
│ медленный, для ключей          │
│ Безопасность: EXCELLENT        │
│ Аппаратное: ✅ Да              │
├─────────────────────────────────┤
│ ✅ ПРЕИМУЩЕСТВА:               │
│ • Асимметричный (обмен)        │
│ • Для цифровых подписей        │
│ • Аппаратное ускорение         │
├─────────────────────────────────┤
│ ⚠️ СЛАБОСТИ:                   │
│ • ОЧЕНЬ медленный              │
│ • Максимум ~400 байт за раз    │
│ • Размер ключей > 4 KB         │
└─────────────────────────────────┘
```

**Объяснение:**
> "RSA используется НЕ для шифрования данных, а для обмена AES ключами. 
> Представь: шифровать 1 MB RSA это 2500+ операций, 250 секунд работы. 
> А AES 1 MB делает за 1-2 миллисекунды. Поэтому практика такая: 
> RSA передает AES-ключ, потом AES шифрует большие данные."

## 💬 ОТВЕТЫ НА ТИПИЧНЫЕ ВОПРОСЫ

**Q: Почему AES-256 лучше чем AES-128?**  
A: 256-bit ключ vs 128-bit. Время перебора: 10^64 лет vs 10^32 лет. AES-256 защита от future квантовых компов.

**Q: DES реально взломан?**  
A: Да. 56-bit ключ перебирается за 24 часа. В 1990-х это был стандарт, сейчас - история.

**Q: Почему нельзя RSA для больших данных?**  
A: Медленно (120ms на операцию) и можно шифровать только ~400 байт за раз. 1 MB = 2500+ операций = 250 сек.

**Q: ChaCha20 vs AES-GCM - что выбрать?**  
A: На современных CPU - AES-GCM. На мобилях - ChaCha20 (нет AES-NI).

**Q: А PLAINTEXT это реально используется где-то?**  
A: К сожалению да... старые системы, legacy code. Твоя задача - избежать.
