Пользователи и доступ

Визуальная демонстрация методов аутентификации, ролей и управления аккаунтом.

Главная Пользователи Личные сообщения Группы Файлы Ключи и аудит

Методы страницы

  • POST /api/users/register - регистрация с генерацией ключей и токена.
  • POST /api/users/{login}/confirm - подтверждение аккаунта.
  • POST /api/users/auth - 2FA: пароль + hardware token.
  • POST /api/users/{login}/role - назначение роли.
  • POST /api/users/{login}/token/rotate - ротация токена.
  • POST /api/users/{login}/block, /recover - блокировка и восстановление.

Выбери действие: после каждого вызова справа видно результат и что именно изменилось.

Действия по сценариям

Сценарий A: Регистрация пользователя

Создает учетную запись, пару ключей и аппаратный токен (если не указан, генерируется автоматически).

Сценарий B: Подтверждение аккаунта

Переводит пользователя в активный статус, после этого доступна успешная авторизация.

Сценарий C: Вход с 2FA

Проверяет одновременно пароль и аппаратный токен.

Сценарий D: Смена роли (RBAC)

Меняет роль пользователя: USER, OPERATOR, ADMIN.

Сценарий E: Админ-действия по аккаунту

Ротация токена, блокировка пользователя и восстановление после компрометации.

Визуальный flow

1. Регистрация
2. Подтверждение
3. Авторизация 2FA
4. Роль и админ-действия

Подсказка: для демонстрации начни строго по шагам 1 -> 2 -> 3, затем показывай шаг 4.
