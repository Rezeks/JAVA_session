Личные сообщения

Показывает полный цикл: отправка -> доставка -> прочтение/ошибка -> история/офлайн.

Главная Пользователи Личные сообщения Группы Файлы Ключи и аудит

Методы страницы

  • POST /api/messages/send - отправка личного сообщения.
  • POST /api/messages/{messageId}/deliver - статус delivered.
  • POST /api/messages/{messageId}/read?readerId=... - статус read.
  • POST /api/messages/{messageId}/error - статус error.
  • GET /api/messages/history/{userId} - история пользователя.
  • GET /api/messages/offline/{userId} - офлайн-пулл.

Отправь сообщение и управлять его статусом через кнопки ниже.

Действия по сценариям

Сценарий A: Отправка личного сообщения

Создает зашифрованное и подписанное сообщение от отправителя к получателю.

Сценарий B: Управление статусом сообщения

Используй `messageId` из шага A: пометь доставку, прочтение или ошибку.

1. SENT
2. DELIVERED
3. READ
Alt: ERROR

Подсказка: обычно показывают путь SENT -> DELIVERED -> READ. ERROR можно показать как альтернативный исход.

Сценарий C: История и офлайн-пулл

Показывает накопленные сообщения пользователя и очередь офлайн-доставки.
