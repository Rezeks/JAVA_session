Передача файлов

Упрощенный локальный сценарий: один chunk, finalize, проверка checksum, delivered.

Главная Пользователи Личные сообщения Группы Файлы Ключи и аудит

Методы страницы

  • POST /api/files/init - открытие transfer с размером файла.
  • POST /api/files/{transferId}/chunk?offset=... - загрузка части.
  • POST /api/files/{transferId}/finalize - шифрование и сохранение.
  • GET /api/files/{transferId}/verify?checksum=... - проверка хеша.
  • POST /api/files/{transferId}/delivered - подтверждение доставки.

Выбери файл и запусти демонстрационный pipeline.

Действия по сценариям

Сценарий A: Подготовка передачи

Выбери отправителя, получателя и файл. Далее backend выполнит init/chunk/finalize/verify/delivered.

Сценарий B: Наблюдение шагов

Ниже визуально отображаются этапы, чтобы показать логику передачи без чтения логов backend.

1. INIT
2. CHUNK
3. FINALIZE

Подсказка: для демо достаточно небольшого файла, он загрузится одним chunk.
