## 🎬 Во время демонстрации

### Шаг 1: Пользователи (1-2 мин)
- [ ] Открыл `http://localhost:8080/ui/users.html`
- [ ] Зарегистрировал alice
- [ ] Зарегистрировал bob
- [ ] Подтвердил (confirm) обоих
- [ ] Provera: возможна 2FA аутентификация

**Что говорить:**
> "Каждый пользователь получает свою пару RSA-2048 ключей. Пароли хешируются PBKDF2 с 100 тысячами итераций - это защищает от перебора."

### Шаг 2: Сообщения (2-3 мин)
- [ ] Открыл `http://localhost:8080/ui/messages.html`
- [ ] Отправил сообщение от alice к bob
- [ ] Скопировал Message ID
- [ ] Подтвердил доставку (deliver)
- [ ] Проверил историю
- [ ] Пулл офлайн сообщений

**Что показывать:**
- Encrypted payload (очень длинная строка base64)
- Wrapped message key (еще одна длинная строка)
- Signature (подпись)

**Что говорить:**
> "Сообщение шифруется AES-256-GCM - это authenticated encryption. Даже если его перехватить, он нечитаемый. OneTimeKey оборачивается в публичный ключ получателя - только он сможет раскрыть. Все подписано - получатель знает, что это точно от отправителя."

### Шаг 3: Групповые чаты (1 мин)
- [ ] Открыл `http://localhost:8080/ui/groups.html`
- [ ] Создал группу
- [ ] Отправил сообщение в группу

**Что говорить:**
> "Групповые чаты: одно сообщение шифруется один раз, но OneTimeKey оборачивается для каждого члена группы в его публичный ключ. Масштабируемо."

### Шаг 4: Файлы (1-2 мин)
- [ ] Открыл `http://localhost:8080/ui/files.html`
- [ ] Инициировал загрузку
- [ ] Загрузил чанк
- [ ] Финализировал
- [ ] Проверил хеш

**Что говорить:**
> "Файлы до 2 ГБ шифруются потоком - не загружаются в память целиком. Можно резюмировать загрузку. SHA-256 хеш позволяет проверить целостность."

### Шаг 5: Аудит (0.5 мин)

- [ ] Показал список всех операций
- [ ] Объяснил: все действия логируются для forensics

## 📋 Возможные проблемы и решения

### Проблема: "API недоступен на 8080"
**Решение:**
- Проверь, что приложение запущено (`mvn spring-boot:run`)
- Дождись вывода "Server started on port 8080"
- Проверь firewall

### Проблема: "Kafka connection error"
**Решение:**
- Запусти Kafka: `.\kafka-console-startup.bat` (или для твоей OS)
- Проверь `localhost:9092` доступен
- Можно продолжить demo без Kafka (события просто не будут логироваться)

### Проблема: "Maven compilation failed"
**Решение:**
- Чистка: `mvn clean`
- Полная пересборка: `mvn -DskipTests clean compile`
- Проверь Java version: `java -version` (должна быть 21+)

### Проблема: "UI не загружается"
**Решение:**
- Проверь console нет ошибок (смотри браузер F12)
- Попробуй refresh страницы (Ctrl+F5 для hard refresh)
- Проверь что сервер запущен и слушает 8080
